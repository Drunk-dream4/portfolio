# 响应式高性能图片相册

## 项目概述
这是一款基于原生 HTML/CSS/JavaScript 开发的轻量级响应式图片相册，专为解决多图、大图场景下页面加载卡顿、渲染缓慢的痛点设计。项目无任何第三方框架依赖，开箱即用，可快速部署到任意静态网页托管平台，兼顾极致性能与完整的相册交互体验。

## 核心特性
- 🚀 **极致加载性能**：内置智能懒加载、WebP 格式兼容、异步图片解码等多重优化，彻底解决大图、多图导致的页面加载卡顿问题
- 📱 **全设备响应式**：基于 CSS Grid 实现自适应布局，完美适配 PC 端、平板、手机等各类屏幕尺寸
- 🎨 **开箱即用**：纯原生实现，零依赖、零配置，仅需替换图片数据即可直接部署使用
- 🖼️ **完整交互体验**：支持图片卡片点击灯箱全屏预览、点击关闭、hover 微动效等完整相册基础功能
- 🧩 **强兼容降级**：针对老旧浏览器做了兼容处理，保证在低版本浏览器中也能正常加载使用

## 快速上手
### 前置准备
1. **图片压缩**：使用 [TinyPNG](sslocal://flow/file_open?url=https%3A%2F%2Ftinypng.com%2F&flow_extra=eyJsaW5rX3R5cGUiOiJjb2RlX2ludGVycHJldGVyIn0=)、ILoveIMG 等工具，将单张图片体积压缩至 200KB-500KB 以内
2. **格式优化**：将 JPG/PNG 图片批量转换为 WebP 格式，保证 WebP 文件与原图片同目录、同文件名（仅后缀不同）

### 部署步骤
1. 复制完整项目代码，新建文件并命名为 `index.html`
2. 打开文件，找到 `const photos = [...]` 配置项，替换为你实际的图片路径、标题与描述
3. 将 `index.html` 与所有图片文件上传至你的网页服务器/静态托管平台
4. 访问对应地址即可正常使用，本地预览可直接双击 `index.html` 文件打开

## 核心优化说明（解决加载卡顿的关键）
1. **智能懒加载机制**：基于 Intersection Observer API 实现，仅当图片进入视口附近时才触发加载，大幅减少初始页面的 HTTP 请求数与带宽占用
2. **图片格式深度优化**：优先加载 WebP 高清低体积格式，同等画质下体积比 JPG/PNG 减少 30%-60%，同时自动降级兼容原图片格式
3. **渲染性能优化**：固定图片渲染高度+占位背景，避免图片加载时的页面布局重排；添加 `decoding="async"` 异步解码属性，图片解码不阻塞页面主线程
4. **滚动流畅度优化**：设置 200px 预加载缓冲区域，图片即将进入视口时提前触发加载，保证滚动浏览的无缝流畅体验

## 自定义配置
### 基础信息修改
- 相册标题：修改 `<h1>` 标签内的文本内容
- 页面标题：修改 `<title>` 标签内的网页标题

### 布局与样式调整
- 网格列数：修改 `.gallery` 选择器中 `grid-template-columns` 的参数，调整卡片最小宽度与列数
- 卡片尺寸：修改 `.photo-card img` 中的 `height` 属性，调整图片卡片的展示高度
- 主题配色：修改对应 CSS 选择器中的颜色、圆角、动效等参数，适配个人设计风格

### 性能参数调整
- 预加载距离：修改 `initLazyLoad` 函数中 `rootMargin` 的数值，调整图片提前加载的距离
- 分批加载：若图片数量超过 100 张，可开启分批加载逻辑，进一步降低初始渲染压力
